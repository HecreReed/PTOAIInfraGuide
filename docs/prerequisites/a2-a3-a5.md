# A2 / A3 / A5 代际差异

## 速查表

| 维度 | A2 (910B) | A3 (910C) | A5 (950) |
|------|-----------|-----------|----------|
| 定位 | 广泛部署训练/推理 | 增强代 | 新一代（指令/同步增强） |
| PTO 支持 | 完整 | 完整 | 完整 + 扩展 |
| 同步 | Event | Event | Event + **BufID** |
| 通信 | TGET/TPUT 等 | 同类 | 增强异步通信等 |
| 工具参数 | `a2a3` / `--pto-arch=a3` 等 | 同左 | `a5` / `--pto-arch=a5` |

> 具体 SoC 名在工程里常见 `Ascend910B1`、`Ascend950` 等，以当前 CANN/工具链文档为准。

## 同步：A5 的 BufID

A3 典型写法是显式 event：

```text
MTE2(tile) → set_flag → wait_flag → VEC(tile)
```

A5 增加基于 buffer id 的程序顺序同步（get_buf / rls_buf），减少事件资源压力，但约束更强（如同一物理 buf id 的 get/rls 不能随意嵌套）。

PTOAS 侧常见开关：

```bash
# A3：传统 insert-sync
ptoas input.pto --pto-arch=a3 --enable-insert-sync -o out.cpp

# A5：bufid sync
ptoas input.pto --pto-arch=a5 --enable-bufid_sync -o out.cpp
```

## 对开发者的实践影响

1. **先选平台字符串**：pypto-lib 的 `-p a2a3|a2a3sim|a5|a5sim`。  
2. **不要混用同步模型假设**：为 A3 手写的 event 图不能假定在 A5 上「自动等价最优」。  
3. **读 kernel 目录分层**：pto-isa 常有 `kernels/manual/a2a3` 与 `kernels/manual/a5`。  
4. **性能数字不可跨代硬比**：同一算法不同代的 Bound 结构会变。

## 选型建议

| 场景 | 建议 |
|------|------|
| 学习指令语义 | CPU-SIM，不绑代际细节 |
| 现网 910B/C | `a2a3` 主线 |
| 新硬件/新特性 | 跟 A5 文档与 `a5` 样例 |
| 写可移植库 | 抽象在 pto-isa 指令层，代际差异关进后端实现 |
